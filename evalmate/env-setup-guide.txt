EvalMate Environment Setup Guide
=================================

1. Create a new file named `.env.local` in the evalmate folder
2. Copy the following content into it:

```
# ========================================
# EvalMate Environment Configuration
# ========================================

# ----------------------------------
# SUPABASE CONFIGURATION
# ----------------------------------
# Get these from: https://supabase.com/dashboard/project/YOUR_PROJECT/settings/api
NEXT_PUBLIC_SUPABASE_URL=your_supabase_project_url
NEXT_PUBLIC_SUPABASE_ANON_KEY=your_supabase_anon_key
SUPABASE_SERVICE_ROLE_KEY=your_supabase_service_role_key

# ----------------------------------
# GROQ AI CONFIGURATION
# ----------------------------------
# Get API key from: https://console.groq.com/keys
GROQ_API_KEY=your_groq_api_key

# ----------------------------------
# RAZORPAY PAYMENT CONFIGURATION
# ----------------------------------
# Get these from: https://dashboard.razorpay.com/app/keys
NEXT_PUBLIC_RAZORPAY_KEY_ID=rzp_test_your_key_id
RAZORPAY_KEY_SECRET=your_key_secret

# ----------------------------------
# NEXT.JS CONFIGURATION
# ----------------------------------
# Generate a random secret: openssl rand -base64 32
NEXTAUTH_SECRET=your_random_secret_key
NEXTAUTH_URL=http://localhost:3000
```

3. Replace the placeholder values with your actual API keys (see instructions below)


DETAILED SETUP INSTRUCTIONS:
============================

SUPABASE SETUP:
1. Go to https://supabase.com and create a free account
2. Create a new project
3. Go to Settings > API
4. Copy the Project URL and paste it as NEXT_PUBLIC_SUPABASE_URL
5. Copy the "anon public" key and paste it as NEXT_PUBLIC_SUPABASE_ANON_KEY
6. Copy the "service_role secret" key and paste it as SUPABASE_SERVICE_ROLE_KEY

DATABASE SETUP:
1. In your Supabase project, go to SQL Editor
2. Copy the entire content of supabase-schema.sql from this project
3. Run the SQL to create tables and RLS policies

GROQ AI SETUP:
1. Go to https://console.groq.com/
2. Create an account and sign in
3. Go to API Keys section
4. Create a new API key
5. Copy the key and paste it as GROQ_API_KEY

RAZORPAY SETUP:
1. Go to https://razorpay.com/ and create an account
2. Complete KYC verification (required for Indian users)
3. Make sure you're in "Test mode" (dashboard will show test keys)
4. Go to Settings > API Keys
5. Generate new API Key Pair if none exist
6. Copy the "Key ID" (starts with rzp_test_) and paste it as NEXT_PUBLIC_RAZORPAY_KEY_ID
7. Copy the "Key Secret" and paste it as RAZORPAY_KEY_SECRET

WEBHOOK SETUP:
1. In Razorpay Dashboard, go to Settings > Webhooks
2. Click "Add New Webhook"
3. For development, use: https://your-supabase-project.supabase.co/functions/v1/razorpay-webhook
4. Select events: payment.captured
5. Save the webhook (Razorpay uses webhook signature verification)

DEPLOYMENT STEPS:
1. Deploy Supabase Edge Functions:
   supabase functions deploy create-razorpay-order
   supabase functions deploy razorpay-webhook

2. Test payments in Razorpay Dashboard under "Test Transactions"

NOTE: Razorpay requires Indian phone numbers for testing. You may need to use a valid Indian mobile number for testing payments.

NEXTAUTH SETUP:
1. Generate a random secret:
   - On Windows: You can use any random string like "my-secret-key-12345"
   - On Linux/Mac: Run: openssl rand -base64 32
2. Paste it as NEXTAUTH_SECRET
3. For development, NEXTAUTH_URL should be http://localhost:3000

SECURITY NOTES:
- Never commit .env.local to version control
- Use test keys for development
- The .env.local file is already in .gitignore
- Keep your service role keys secure - they have admin access
